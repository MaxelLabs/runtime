## 调查报告类型

### 🔍 问题调查 (Problem Investigation)
针对具体技术问题的深入分析：
- 问题描述和根因分析
- 影响范围评估
- 解决方案建议
- 预防措施

**示例**: matrix4-api-investigation.md

### 📊 技术盘点 (Technology Survey)
对现有技术栈和工具的全面盘点：
- 现有资源清单
- 技术成熟度评估
- 缺失项识别
- 改进建议

**示例**: basic-rendering-demos-investigation.md

### 🚀 可行性研究 (Feasibility Study)
新技术的可行性分析和实现路径：
- 技术依赖分析
- 实现难度评估
- 风险识别
- 实施建议

**示例**: lighting-material-investigation.md

## 使用指南

### 🎯 如何使用调查报告

1. **技术选型阶段**:
   - 查看相关的技术调查报告
   - 了解现有技术栈的成熟度
   - 识别潜在的技术风险

2. **问题诊断阶段**:
   - 参考问题调查报告
   - 了解已知问题的根因和解决方案
   - 避免重复遇到相同问题

3. **规划制定阶段**:
   - 基于调查结果制定实施策略
   - 评估工作量和时间计划
   - 识别依赖关系和风险

### 📊 调研质量评估

每份调查报告都包含：

- **完整性**: 覆盖所有相关技术点
- **准确性**: 技术信息准确可靠
- **实用性**: 提供具体可行的建议
- **时效性**: 反映最新技术状态

### 🔄 报告更新机制

- **定期更新**: 每月审查技术栈变化
- **事件驱动**: 技术升级或重大变更后更新
- **反馈驱动**: 基于实施反馈补充内容

## 调研方法论

### 📋 调研流程

1. **明确调研目标**
   - 定义具体的调研问题
   - 确定调研范围和深度

2. **信息收集**
   - 代码库扫描和分析
   - 文档和资料查阅
   - 技术专家咨询

3. **分析整理**
   - 现状梳理和分类
   - 问题识别和根因分析
   - 依赖关系梳理

4. **建议制定**
   - 实现路径设计
   - 风险评估和缓解
   - 时间和资源估算

5. **报告编写**
   - 结构化内容组织
   - 详细的证据支持
   - 明确的行动建议

### 🔍 分析方法

- **SWOT分析**: 优势、劣势、机会、威胁
- **成熟度模型**: 技术成熟度分级评估
- **依赖分析**: 技术依赖关系图
- **风险评估**: 技术、时间、资源风险
